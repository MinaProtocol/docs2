# Hard Fork Upgrade

A hard fork occurs when there are major changes to the core protocol that require all nodes on the network to update to the latest software. To learn more, see [Hard Forks](/mina-protocol/hardforks).

## How to prepare for Berkeley hard fork

The Berkeley hard fork upgrades all nodes, including archive nodes. One of required steps is to migrate
archive databases from the current Mainnet format to Berkeley. This migration requires actions and efforts from
node operators and exchanges. 

Read more about archive data migration:

- [How to prepare for archive migration](/hard-fork-upgrade/prerequisites-and-planning)
